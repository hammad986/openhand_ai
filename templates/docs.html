Quick Start

Get Nexora running in under 2 minutes.

1. Clone & Install

git clone https://github.com/your-org/nexora-ai
cd nexora-ai
pip install -r requirements.txt

2. Configure Secrets

Add these in Replit Secrets or as environment variables:

SECRET_KEY=your-flask-secret
JWT_SECRET=your-jwt-secret
# Optional — for AI models
OPENAI_API_KEY=sk-...
GEMINI_API_KEY=...
# Optional — for billing
RAZORPAY_KEY_ID=rzp_live_...
RAZORPAY_KEY_SECRET=...
EMAIL_API_KEY=re_...   # Resend

3. Run

python web_app.py

Open http://localhost:5000 in your browser. The platform initialises all databases automatically on first run.

✓ The platform uses SQLite by default — no external database setup required.

Architecture

Backend

  • web_app.py — Flask application (~10,000 lines), all API routes, business logic
  • agent.py — Core agent loop and task execution engine
  • orchestrator.py — Multi-agent orchestration and pipeline management
  • router.py / model_router.py — Intelligent LLM routing
  • memory.py — 3-tier memory system (short, long, semantic)
  • payments.py — Razorpay billing integration
  • auth_system.py — JWT authentication
  • tools.py + tools/ — Extensible tool registry

Frontend

  • templates/index.html — Single-page application (~16,000 lines) with all UI phases
  • Three-panel layout: AI Thinking (left), Logs/Code/Terminal (centre), Inspector (right)
  • Real-time WebSocket-like polling for agent status and queue updates

Databases

  • sessions.db — Sessions, settings, scheduler tasks, chat history
  • billing.db — Subscriptions, invoices, payment events

Request Flow

Browser → Flask (web_app.py) → Agent Loop → Model Router
                                         ↓
                               Tool Execution (tools/)
                                         ↓
                               Memory System (memory.py)
                                         ↓
                               Response → Browser

Specialist Agents (Phase 7)

  • Code Reviewer — Static analysis, best practices, code quality
  • Debugger — Root cause analysis, error tracing, fix suggestions
  • Test Generator — Unit tests, integration tests, coverage analysis
  • Security Auditor — Vulnerability detection, OWASP checks
  • Performance Optimizer — Bottleneck identification, optimization suggestions

Model Routing System

Phase 9 assigns specific models for each task role based on plan tier and user preference.

Routing Modes

ModeDescriptionUse Case
LiteCost-optimised, fast modelsQuick tasks, prototyping
ProBalanced cost/qualityGeneral development
EliteBest models, highest qualityComplex architecture, critical code

Priority Settings

  • Cheapest — Minimise cost, use free tiers
  • Fastest — Minimise latency, prefer Groq / Flash models
  • Smartest — Maximise quality, prefer GPT-4o / Claude 3.5 / Gemini Ultra

Auto-Failover

If the primary provider fails or rate-limits, the router automatically falls back to the next best available provider with no user intervention required.

Memory System

Three-Tier Architecture

  • Tier 1 — Short-term: In-session messages and context (full detail)
  • Tier 2 — Mid-term: Compressed summaries of older context (intelligently summarised)
  • Tier 3 — Long-term: Semantic vector store via ChromaDB (cross-session recall)

Self-Correction

The system tracks errors and successful patterns. On each new task, relevant past failures are surfaced to the agent to prevent repeating mistakes (Phase 14).

Intelligence Score

A per-session grade (A–F) based on success rate, error rate, and task complexity — visible in the Inspector panel.

Multi-Agent System

Collaboration Pipeline (Phase 11)

Complex tasks are automatically decomposed across 4 specialised agents:

  1. Manager Agent — Decomposes the goal, assigns sub-tasks, tracks progress
  2. Research Agent — Gathers context, documentation, best practices
  3. Coding Agent — Implements the solution, writes code
  4. Debug Agent — Tests, finds bugs, ensures correctness

Autonomous Goals (Phase 16)

Users can define high-level goals. The system automatically decomposes them into a DAG of sub-tasks, executes them in dependency order, and reports progress in real time.

Scheduler System

Phase 18 adds a background task scheduler for autonomous agent execution.

Capabilities

  • Schedule tasks at specific times or intervals (cron-like)
  • Run AI agents in the background without user interaction
  • Email notifications on completion (requires EMAIL_API_KEY)
  • Task history and status tracking

Use Cases

  • Nightly code reviews and test runs
  • Automated documentation generation
  • Periodic security audits
  • Scheduled data processing pipelines

BYOK System (Bring Your Own Key)

Phase 5 allows users to supply their own API keys for any supported provider.

How It Works

  1. Open the Provider Settings panel in the platform
  2. Enter your API key for any provider (OpenAI, Anthropic, Groq, etc.)
  3. Keys are stored encrypted in the session database
  4. The router automatically uses your keys before falling back to platform keys
💡 BYOK allows unlimited usage on any provider — costs are billed directly to your account by the provider.

Billing Setup (Razorpay)

⚠ Billing requires a Razorpay account. Get your keys at razorpay.com

Step 1 — Add Secrets

RAZORPAY_KEY_ID=rzp_live_xxxxxxxx
RAZORPAY_KEY_SECRET=xxxxxxxxxxxxxxxx
EMAIL_API_KEY=re_xxxxxxx   # Optional — Resend for invoice emails
EMAIL_FROM=billing@your-domain.com

Step 2 — Configure Webhook

In Razorpay Dashboard → Settings → Webhooks, add:

https://your-domain.replit.app/api/payments/webhook

Enable events: payment.captured, payment.failed

Step 3 — Verify

Open the platform → Click your plan badge → The Billing Setup Guide shows "● Connected" when Razorpay keys are active.

Payment Flow

User clicks "Pay" → /api/payments/create-order (Razorpay order created)
                   → Razorpay Checkout popup opens
                   → User pays → /api/payments/verify (HMAC signature check)
                   → Subscription activated → Invoice generated → Email sent
                   → Razorpay webhook → /api/payments/webhook (idempotent backup)

Security

Authentication

  • JWT-based session authentication (Phase 19)
  • bcrypt password hashing
  • Token expiry and refresh

API Security

  • Rate limiting on all public endpoints
  • Input sanitization and SQL injection prevention
  • Razorpay webhook HMAC signature verification
  • CORS headers with configurable origins
  • Emergency kill switch (/api/admin/kill-switch)

Data Security

  • API keys stored in encrypted session storage
  • No plaintext secrets in logs
  • WAL mode SQLite for crash-safe writes
  • Structured logging with no sensitive data leakage

Deployment Guide

Replit (Recommended)

  1. Fork or import the project to Replit
  2. Add all required secrets in Replit's Secrets panel
  3. Click Deploy to publish to *.replit.app
  4. Your app gets TLS, health checks, and auto-restart automatically

Self-Hosted (Gunicorn)

gunicorn -w 2 -b 0.0.0.0:5000 web_app:app

Production Checklist

  • ✓ Set SECRET_KEY and JWT_SECRET to long random strings
  • ✓ Use Razorpay live keys (not test keys)
  • ✓ Configure webhook URL with your production domain
  • ✓ Set EMAIL_FROM to a verified sender domain
  • ✓ Enable HTTPS (automatic on Replit)
