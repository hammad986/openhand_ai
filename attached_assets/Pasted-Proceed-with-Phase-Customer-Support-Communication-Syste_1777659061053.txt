Proceed with Phase: Customer Support & Communication System (Production Grade).

This is a critical trust layer for SaaS. Build it with enterprise-level reliability.

---

# 🎯 OBJECTIVE

Create a full support system that allows:

* Users to raise issues
* Admin to track/respond
* Email notifications for trust
* Ticket lifecycle management

---

# 🧱 PART 1 — DATABASE DESIGN

Create table: support_tickets

Fields:

* id (UUID)
* user_id
* subject
* message
* status (open / in_progress / resolved / closed)
* priority (low / medium / high)
* created_at
* updated_at

Create table: ticket_messages

* id
* ticket_id
* sender (user/admin)
* message
* created_at

---

# 📩 PART 2 — API ENDPOINTS

Implement:

POST /api/support/ticket
GET /api/support/tickets
GET /api/support/ticket/<id>
POST /api/support/ticket/<id>/reply
PATCH /api/support/ticket/<id>/status

---

# 📧 PART 3 — EMAIL SYSTEM

Integrate email provider (Resend or SMTP):

On ticket creation:

* send confirmation email to user

On reply:

* notify user

On status change:

* notify user

Email must include:

* ticket ID
* subject
* message preview

---

Create Support tab in UI:

User side:

Admin side (basic):

* ticket list
* status change
* reply panel
